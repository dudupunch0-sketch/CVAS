CVAS PROJECT DOSSIER

C-model code를 hardware-oriented datapath insight로 바꾸는 분석 파이프라인

CVAS는 CVAS_START ~ CVAS_END 구간의 C-model을 읽고, 함수 구조, 데이터 흐름, 제어 흐름, 호출 그래프, cycle 추정, 시각화 정보를 JSON IR과 HTML Viewer로 정리하는 도구다.

왜 필요한가

ISP나 HLS 전처리 단계의 C-model은 연산 자체보다 데이터 의존성, 제어 분기, 함수 호출 관계를 읽는 비용이 더 크다. CVAS는 이 비용을 줄여서 datapath 설계와 pipeline 판단에 필요한 사실만 드러낸다.

Core Problem 복잡한 C-model에서 datapath, control, dependency를 수작업으로 해석해야 한다.
Core Solution 정적 분석 중심 파이프라인으로 block / operation / signal / call graph를 추출한다.
Primary Output CLI 자동화에 바로 쓸 수 있는 machine-readable JSON contract와 operation-flow data.
Secondary Output Operation flow, control flow, function flow를 읽을 수 있는 standalone HTML viewer.

핵심 포인트

Hardware Target II = 1
Analysis Focus Datapath + Control
Output Contract JSON + HTML
Execution Mode CLI-first

동작 흐름

입력 C-model이 어떤 단계를 거쳐 hardware-friendly 정보로 바뀌는지 한눈에 보이도록 정리한 흐름이다.

Step 1

Input

C source에서 분석할 구간만 지정한다. CVAS는 전체 컴파일러가 아니라, hardware 해석에 필요한 구간을 안정적으로 뽑는 데 집중한다.

C-model CVAS_START / CVAS_END project mode
Step 2

Static Analysis

expression lowering, assignment normalization, CFG, call graph, unresolved symbol, rough cycle estimation을 만든다. 이 단계가 사실의 기준이다.

operations signals CFG call graph
Step 3

IR Assembly

분석 결과를 block / operation / signal / flow 구조로 합쳐 CLI와 viewer가 공유하는 JSON contract로 직렬화한다.

blocks operations signals flow
Step 4

Outputs

자동화용 JSON과 사람 검토용 HTML viewer를 생성한다. 필요하면 function_io와 LLM 보강 단계도 후속으로 붙일 수 있다.

output.json output.html function_io.json

구조와 책임 분리

현재 리팩터링 방향은 CLI 진입점, 분석 파이프라인, 함수 단위 패스, viewer 소비 계층을 분리하는 것이다.

실행 계층

  • src/cvas_mvp.py : 호환용 wrapper
  • src/cvas_cli.py : CLI argument parsing / file I/O / execution
  • cvas_wrapper.py : JSON 생성 후 HTML viewer까지 한 번에 실행

분석 계층

  • src/cvas_pipeline.py : orchestration, model assembly
  • src/cvas_passes.py : function-level preprocessing / lowering / per-function analysis
  • src/cvas_expr.py, src/cvas_cfg.py, src/cvas_callgraph.py : 세부 분석 모듈

계약 계층

  • src/cvas_model.py : IR dataclass와 cycle rules
  • src/cvas_serialize.py : JSON serialization
  • 핵심 필드: blocks, operations, signals, flow

시각화 계층

  • json_to_html.py : standalone HTML viewer 생성
  • Diagram 탭: block / signal / operation structure
  • Sequence 탭: 함수 호출 순서, dependency hint, function_io overlay

주요 명령어

프로젝트 소개 자료에서 바로 복붙 가능한 형태로 정리한 기본 실행 시나리오다.

1. 기본 분석

C-model을 읽고 JSON 분석 결과를 만든다.

python src/cvas_cli.py model.c -o output.json

2. HTML Viewer 생성

생성된 JSON을 operation-flow 중심 Diagram 탭과 function-flow Sequence 탭이 포함된 단일 HTML로 바꾼다.

python json_to_html.py output.json output.html

3. End-to-End 실행

분석부터 HTML viewer 생성, ELK asset 복사까지 한 번에 처리한다.

python cvas_wrapper.py test_examples.c docs/test_examples_output.html \
  --output-json docs/test_examples_output.json

4. function_io 생성

Sequence 탭 dependency lane 품질을 높이기 위한 함수별 reads / writes 메타데이터를 만든다.

python tools/generate_function_io.py model.c --llm-provider none

5. Codex CLI 기반 보강

개발 환경에서 Codex CLI를 이용해 function_io를 refine / verify 한다.

python tools/generate_function_io.py model.c \
  --llm-provider codex-cli \
  --codex-danger-full-access \
  --codex-timeout-sec 60

6. Quick Syntax Check

핵심 실행 파일의 문법 오류를 빠르게 확인한다.

python -m py_compile \
  src/cvas_mvp.py \
  src/cvas_cli.py \
  src/cvas_pipeline.py \
  src/cvas_passes.py \
  json_to_html.py \
  tools/generate_function_io.py

사용 시나리오

이 프로젝트를 소개할 때 가장 이해하기 쉬운 대표 사용 흐름이다.

Hardware Review

기존 C-model의 함수 경계, 데이터 이동, 제어 분기를 빠르게 읽어야 할 때 쓴다.

  • datapath 후보 파악
  • critical path 확인
  • control-heavy 함수 식별

CLI Automation

배치 실행으로 JSON을 뽑고, 후속 도구나 agent가 그 결과를 읽도록 만들 수 있다.

  • file-oriented contract
  • project mode indexing
  • deterministic output baseline

Future Roadmap

현재 설계 방향은 II=1 pipeline 해석과 hazard 모델, LLM provenance 계층 추가다.

  • pipeline candidates
  • memory / control hazards
  • LLM enrichment with static precedence
핵심 메시지: CVAS는 단순한 C 파서가 아니라, hardware 설계 관점에서 C-model을 재구성하는 분석 파이프라인이다. 최종 목표는 datapath-centric pipeline architecture를 더 빨리 이해하고, 더 안정적으로 자동화 가능한 형태로 정리하는 것이다. 최신 소개 문서는 docs/cvas_project_overview.html, 최신 샘플 viewer 산출물은 docs/test_examples_output.html / docs/test_examples_output.json에 둔다.
